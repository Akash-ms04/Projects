Nike Retail Sales Analysis Using SQL

Project Overview
This project analyzes Nike retail sales data using SQL. The dataset contains transaction level sales records including retailer information, product details, location data, and financial metrics.

The data was cleaned and normalized into relational tables to support structured analysis. SQL queries were used to evaluate retailer performance, product demand, and regional sales trends.

Dataset
Nike retail sales dataset containing more than 10000 transaction records.

Key Columns

Retailer
Retailer ID
Invoice Date
State
City
Product
Units Sold
Price per Unit
Total Sales
Operating Cost
Operating Profit
Operating Margin

Database Design

The dataset was normalized into four tables.

Retailers
Locations
Products
Sales Transactions

This structure reduces redundancy and improves query performance.

Business Questions

Which retailers generate the highest revenue
Which products sell the most units
Which states generate the highest sales
Which products generate the highest profit
How sales change across quarters
Which retailers perform best in each state
